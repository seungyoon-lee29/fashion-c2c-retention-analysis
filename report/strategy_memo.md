# 전략 메모: 활성화 플레이북 + AI/규칙/실험 분별

> 결과(리텐션=재방문): MerRec 첫 구매는 즉시적(획득)이라 리텐션 타깃이 못 됨 → 스파인은 엠바고 후 **재방문**을 모델링.

## 1. 무엇을 발견했나
- 첫 윈도우 **cart 행동(아하 레버)** 은 리텐션(재방문)과 **큰 단순 연관**(naive Δ=+0.151; 카트 유저 재방문 47.7% vs 32.7%).
- 그러나 인게이지먼트(세션수·활동일수·조회수)로 **보정하면 효과가 사라진다**: g-계산 Δ≈**−0.01**, IPTW +0.03 — 둘 다 0의
  ±3pp 안(절대차 3.9pp)이라 **효과 null·부호 미식별**(326% 상대차는 null-스케일 착시이지 진짜 발산 아님). **E-value=1.23**(거의 1).
- **결론: cart는 리텐션 *레버*가 아니라 이미 인게이지된 유저의 *마커*다.** 큰 연관이 인과 보정에서 ~0으로 붕괴한 **허영지표** 사례.
- 진짜 리텐션 드라이버(SHAP)는 **인게이지먼트 폭**(세션수·활동일수·조회수). 검증이 헛돌지 않음: retain-hazard 시간외 PR-AUC=0.184
  (base 0.08의 2배+), embargo gap sweep PR-AUC ~0.19 plateau(누수 통제).

## 2. 팀에 제안하는 방향
- **"첫 cart 유도"를 리텐션 레버로 밀지 마라** — 인과근거 없음(허영지표). cart 넛지는 *즉시 구매(획득)* 엔 의미 있을 수 있으나
  리텐션 효과로 보고하면 Goodhart에 빠진다.
- 리텐션을 원하면 **초기 인게이지먼트 폭**(여러 세션·여러 카테고리 탐색)을 키우는 온보딩에 투자 — 단 이것도 *연관* 우위이므로
  레버로 확정하려면 A/B 필요.
- 임팩트 가늠: 신규 1,000명당 기대 재방문수(Markov ≈357/1000, 레버 원장 범위)로 보고. **단 g-계산 레버 효과≈0이므로 레버 원장은
  상한 시나리오로만** 해석.

## 3. AI를 언제 쓰고 언제 규칙으로 가나 (분별)
- **발굴·개인화 = 모델**: 어떤 행동이 활성화 신호인지(SHAP), 누구에게 무엇을 추천할지(시퀀스 모델, Tier 2)는 모델의 일.
- **운영 = 단순 규칙**: "첫 윈도우 cart ≥ k" 같은 아하 임계는 **규칙으로 운영**(모델 불필요, 투명·저비용·감사가능).
  모델은 규칙을 *찾는* 데 쓰고, *돌리는* 데는 규칙을 쓴다.
- **인과 주장 = 실험**: g-계산은 *조건부* 추정(가정 의존). 진짜 효과 확정과 배율 잠금은 **A/B 테스트**로 — 본 분석은
  "A/B로 검증할 우선순위 가설"을 제공하는 단계.

## 4. 주의 (Goodhart·과적합)
- "활성화 지표를 올린다"가 지표 뒤 실체를 올리는지 측정값만 올리는지 구분. 임계 과적합은 시간외 검증으로 통제.
