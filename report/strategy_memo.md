# 전략 메모: 활성화 플레이북 + AI/규칙/실험 분별

## 1. 무엇을 발견했나
- 신규-관측 유저의 **첫 윈도우 cart 행동(아하 레버)** 이 전환 위험을 끌어올린다. 단순 연관(naive Δ=+0.052)에는
  교란(고친화 유저가 cart도 많이·전환도 많이)이 섞여 있고, **g-계산으로 보정하면 Δ≈+0.040** 으로 줄지만 여전히 양(+).
- 독립 모형(IPTW/MSM)도 +0.050으로 일치 → 단일 모형 의존이 아님. **E-value=2.68**: 이 효과를 설명해 없애려면
  미관측 교란이 관측된 어떤 요인보다도 그만큼 강해야 한다.

## 2. 팀에 제안하는 방향
- **온보딩에서 첫 cart를 유도**(첫 세션 내 위시리스트/장바구니 넛지, 관심 카테고리 추천 배치)하여 활성화율을 올린다.
- 목표는 "전환을 직접 강매"가 아니라 **초기 행동(활성화)** 을 만드는 것 — 생존 곡선상 이탈 위험을 낮추는 지점.
- 임팩트 가늠: 활성화율 Δ당 **신규 1,000명당 기대 전환수**(레버 원장의 보수/기준/낙관)로 보고 → 의사결정에 범위로 제시.

## 3. AI를 언제 쓰고 언제 규칙으로 가나 (분별)
- **발굴·개인화 = 모델**: 어떤 행동이 활성화 신호인지(SHAP), 누구에게 무엇을 추천할지(시퀀스 모델, Tier 2)는 모델의 일.
- **운영 = 단순 규칙**: "첫 윈도우 cart ≥ k" 같은 아하 임계는 **규칙으로 운영**(모델 불필요, 투명·저비용·감사가능).
  모델은 규칙을 *찾는* 데 쓰고, *돌리는* 데는 규칙을 쓴다.
- **인과 주장 = 실험**: g-계산은 *조건부* 추정(가정 의존). 진짜 효과 확정과 배율 잠금은 **A/B 테스트**로 — 본 분석은
  "A/B로 검증할 우선순위 가설"을 제공하는 단계.

## 4. 주의 (Goodhart·과적합)
- "활성화 지표를 올린다"가 지표 뒤 실체를 올리는지 측정값만 올리는지 구분. 임계 과적합은 시간외 검증으로 통제.
