# 아하 모먼트(활성화) 발굴 → 임팩트 추정 — 그로스 데이터 분석

**"목표 Y를 위해 어떤 행동 X를, 윈도우 N 안에, 몇 번 시켜야 하나"** 를 데이터로 찾고, 그걸 올리면 회사가 얼마나 나아가는지까지 정량화하는 그로스 분석 프로젝트.

> 🚧 **기획 단계** — 상세 설계는 [`PLAN.md`](PLAN.md). (구현 전 클라우드에서 기획 정제 중)

## 분석 질문
신규 유저를 **무슨 행동으로 활성화**시켜야 전환(구매)과 리텐션이 오르나? 그 활성화 지표를 올리면 비즈니스 임팩트는 얼마인가?

## 데이터
[MerRec](https://huggingface.co/datasets/mercari-us/merrec) — Mercari가 공식 배포한 C2C 마켓플레이스 **행동 이벤트 로그**(KDD 2025). `item_view → item_like → add_to_cart → offer → buy_comp` 풀퍼널 + 유저·세션·시계열. (인증 불필요, 월 단위 샤딩이라 일부만 받아 경량 분석)

## 접근 (요약 — 상세는 PLAN.md)
1. **북극성/목표 Y** — 주: 구매 전환(`buy_comp`), 보조: 초기 리텐션(재방문). 조작적 정의.
2. **후보 행동 X** — 첫 윈도우 내 조회·좋아요·장바구니·세션수·카테고리 다양성 등.
3. **SHAP**로 Y를 끌어올리는 행동 선별.
4. **아하 모먼트 임계 (k, n)** — "X ≥ k in window n"을 precision/recall/F1/**MCC**·lift로 정당화.
5. **검증** — train/holdout 분리 + 코호트 곡선.
6. **임팩트 추정 = 마르코프 체인** — 유저 상태 전이로 "아하 달성률 Δ → 활성·전환 증가" 시뮬레이션.
7. **전략 제안 + AI 사용 분별** — 팀 방향·실행안, 그리고 *언제 모델을 쓰고 언제 단순 규칙으로 가는지*.

## 정직성 규율
관측 데이터 → 인과 아님("A/B로 검증할 가설"로 명시) · 선택/생존 편향 노출 · 가정은 config 분리 + 민감도.
