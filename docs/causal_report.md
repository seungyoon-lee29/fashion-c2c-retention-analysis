# 식별 가능성 지도 — "보이는 레버가 인과인가" (M2, 자동 생성)
> **BLUF**: 초기 행동 레버 4종 어느 것도 **신뢰할 양성 리텐션 효과를 주지 못한다** — 두 가지 방식으로. 겹침이 충분한 레버(2종)는 보정 시 효과가 **~0/음전**(허영지표 확인); 신호가 큰 레버(2종)는 **겹침 부족으로 식별 불가**. ⇒ 관측 보정의 한계 = **A/B가 답**.
> 데이터 = `data/events.parquet`(person-period) · 특징창 **W=3일**(config 단일출처) · 대상 22,165명 · retain base 40.1% · `make causal` 재계산. (겹침-부족 레버 *집합*은 W에 견고하나 점추정은 W에 따라 변하므로 특정 W 수치는 단정하지 않음.)
## 레버별 식별 진단
_용어: **propAUC**=활동량만으로 처치를 맞히는 정확도(↑=처치가 활동량에 종속) · **비겹침%**=처치확률이 0.05~0.95 밖인 유저 비율(↑=비교할 쌍이 없음 = overlap 실패) · **기준 SMD**=처치·대조 baseline 격차(보정 *필요* 신호) · **naive→IPTW**=보정 전후 효과. 겹침 판정 = `비겹침% ≤ 10%`._

_naive·IPTW는 연관/진단값이다. 겹침 충분 레버에서도 같은 초기창에서 처치와 교란을 함께 재므로 IPTW는 확정적 인과점추정이 아니라 방향성·민감도 진단으로 읽는다. 겹침 부족 레버에선 외삽이라 점추정 철회. E-value는 *naive(연관) RR* 기준(겉보기 연관의 강건성) — `impact_report.md`의 카트 *g-formula* RR E-value(1.23, 보정 후)와는 추정량이 다름._

## 읽기 — 두 가지 실패, 같은 결론
비유: 인과 보정 = 처치 유저마다 '똑 닮은 비처치 쌍둥이'를 찾아 비교. 쌍둥이가 없으면(겹침 부족) 보정은 데이터 밖 *외삽*이다.
- **겹침 충분 → 허영지표 확인**: `cart≥1` +16→+3pp, `cat≥3` +25→-6pp. 겹침이 있어 외삽 위험은 낮지만, 그 보정값마저 **0 근처/음수**로 무너진다 — 동시창 교란의 과대보정 가능성을 감안해도 *겉보기 양성을 뒷받침할 근거가 없다*. (카트 전체 g-formula 분해: `docs/impact_report.md`.)
- **겹침 부족 → 식별 불가**: `offer≥1`(비겹침 68%), `like≥3`(비겹침 56%). 활동량이 처치를 거의 결정해 비교할 쌍이 없다 → naive가 커 보여도 *인과 판정 자체가 불가*(신호가 크다고 인과 아님).

## 한계 (정직)
- 교란을 처치와 **같은 초기창**에서 측정 — 세션수·활동일수는 레버를 거의 결정하므로 이를 조건부에 넣으면 겹침 부족이 *부분적으로 자초*된다(과대보정). 즉 '겹침 부족'은 데이터 한계 + 설계상 동시성의 혼합. 그래도 결론(관측만으론 신뢰할 양성 효과 추정 불가)은 견고하다.
- 식별 가정(순차교환가능성·positivity·consistency)은 검증 불가한 가정 — 이 표는 그중 **positivity/overlap**만 진단.

## 다음 단계 (1개) — A/B
> 겹침 충분한 레버는 효과가 ~0, 신호 큰 레버는 식별 불가 → **어느 관측 경로도 양성 리텐션 효과를 확정하지 못한다.** 답은 관측 보정 고도화가 아니라 **노출 무작위화(A/B)** — 설계는 `docs/ab_test_design.md`.

> Honesty: positivity/overlap 진단이며 인과 점추정 아님. 겹침 실패 레버는 인과 단정을 철회하고 연관+E-value로만 본다(프로젝트 규약).
