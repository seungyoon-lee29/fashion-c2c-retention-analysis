# A/B 설계 — 첫날 경험 폭 넓히기 넛지 -> D7 재방문 (M3, 자동 생성)

> **BLUF**: 관측 로그만으로는 양성 리텐션 효과를 확정할 수 없다. 따라서 첫날 탐색 폭 넛지를 유저 단위 A/B로 검증한다. 주지표 D7 재방문(base **50.1%**), MDE **2.0pp**, 표본 **9,805명/arm**, 예상 **4.7주**.

## 1. 왜 실험인가

- 코호트 신호: 첫날 활동량 상위군은 하위군보다 D7 재방문이 높다.
- 식별 진단: 카트와 탐색 폭은 좋아 보이지만 같은 초기창 활동량을 보정하면 효과가 0 근처 또는 음수로 불안정하다.
- 결론: 관측 추정치를 더 단정하지 않고, 전향 무작위화로 하나의 실험 estimand를 새로 정의한다.

## 2. 가설

- **H1**: 신규 유저의 첫 세션에서 탐색 폭을 넓히는 넛지가 D7 재방문을 높인다.
- 조회 직후 찜 후보와 인접 카테고리 노출은 별도 실험이 아니라 같은 탐색 폭 넛지의 구현 예시다.

## 3. 설계

- **랜덤화 단위**: 유저. 첫 세션 진입 시 1회 배정하고 고정한다.
- **대상 모집단**: 신규 관측 유저. 프로덕션에서는 가입 후 첫 세션 유저로 고정하는 것이 더 좋다.
- **처치**: 첫 세션에 인접 카테고리와 저장 후보를 노출한다.
- **주지표**: D7 재방문율.
- **보조지표**: first-session categories, like reach, view depth, D1 session count.
- **가드레일**: 구매완료율, 즉시이탈, 세션 품질, SRM.

## 6. 성공/중단 판정표

| Decision | Rule |
|---|---|
| Ship / scale | D7 재방문 lift >= 2.0pp이고 가드레일 악화 없음 |
| Iterate | D7 lift가 0~2.0pp 사이이며 보조지표가 개선됨 |
| Stop | D7 재방문 lift <= 0 또는 즉시이탈 +1.0pp 초과 |
| Invalidate | SRM p<0.001, 로깅 결측, 처치 오염 발견 |
| Revenue follow-up | D7 lift가 확인되면 D14/D30 재방문과 repeat purchase 또는 GMV/AOV 로그로 후속 검증 |

Guardrail margin은 실제 서비스 리스크 허용도에 맞춰 사전등록해야 한다. 현재 데이터에서는 구매완료율이 4.4%로 낮아 작은 구매 악화는 underpowered일 수 있으므로, 구매완료율은 단독 stop rule보다 모니터링 가드레일로 둔다.

## 7. 실험이 답하는 것 / 못 하는 것

- **답함**: 첫날 탐색 폭 넛지의 D7 재방문 인과 효과.
- **못함**: 장기(>D7) 효과, 매출 효과, 구현 방식별 차이, 신규 외 유저 효과.

> Honesty: 가설, 주지표, MDE, 성공/중단 규칙은 실험 전 사전등록 대상이다. 사후 지표/MDE 변경은 금지한다.
