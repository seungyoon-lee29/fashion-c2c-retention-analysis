# 후보 C-fixed — 하이브리드 (B 서사 겉 + A 식별지도 깊이) ★정본 후보★
> **패션 C2C 그로스 분석 — 퍼널·리텐션을 읽고, "보이는 레버가 진짜인지" 인과로 검증하고, 다음 실험을 설계한다 (DA 포트폴리오)**

## 정체성
"행동로그로 **퍼널 누수를 찾고, 리텐션 드라이버를 검증하고, 다음 실험을 설계하는** 프로덕트 데이터 분석가.
*겉보기 좋은 지표(카트·찜)에 속지 않고* 진짜 북극성(재방문)을 움직이는 행동만 골라낸다."

> 설계 원칙: **겉(60초 스캔)은 DA-legible, 속(딥다이브)은 인과 엄밀성.** 퍼널·코호트는 누구나 그리지만,
> 그 위에 **식별 가능성 지도 + 실험 설계**까지 얹는 게 이 작품의 해자다.

## 데이터 (단일 출처)
MerRec(Mercari C2C 행동로그) 5파티션·277만 이벤트·43k 유저·49만 세션·31일. 가입 필드 없음(→ t0=첫 관측).

## 분석 척추 (4단계 — 앞 2개=온램프, 3·4=해자)

### 1. 퍼널 — "어디서 새는가" (온램프)
- 도달률 view 98.7 → like 52.6 → cart 22.9 → offer 17.1 → **buy 4.4%**. 단계별 전환·절대 드롭오프 + 세그먼트별(카테고리·세션길이·신규 vs 재방문).
- 산출 `docs/funnel_report.md`. **인사이트**: 전환 극희소(4.4%) → 전환을 KPI로 쓰면 underpowered. (3단계 복선.)
- **[F5] 양성 권고 필수**: "어느 단계 드롭오프를 어느 세그먼트에서 칠지" *구체적 액션 1개*를 반드시 산출(제네릭 탈출).

### 2. 코호트 리텐션 — "누가 돌아오는가" (온램프)
- 첫 관측일 코호트 × 재방문(D1·D3·D7) 매트릭스 + 히트맵. 드라이버 = **인게이지먼트 폭**(세션수·활동일수·조회수), 단일 행동 아님(SHAP, `drivers_report.md` 확인).
- 산출 `docs/cohort_report.md`. **인사이트**: 리텐션은 "한 번의 결정적 행동"이 아니라 폭에서 온다.
- **[F5] 양성 권고 필수**: "폭(예: 첫 세션 카테고리 다양성)을 넓히는 온보딩 넛지" 같은 *육성 가능한 레버 1개* 제시.

### 3. 식별 가능성 지도 — "보이는 레버가 진짜 레버인가" ★핵심 해자★
- 카트=리텐션 레버처럼 보임(카트 유저 재방문 47.7% vs 32.7%, **+15pp 연관**). 인과 보정(g-formula/IPTW)에서 **~0 붕괴**
  (🔬 `make`-재현, W=3: naive +0.151 → g −0.012 → IPTW +0.027, E-value 1.23 — `docs/impact_report.md`).
- **확장(차별화 핵심)**: 레버 4종(cart·offer·탐색폭·like≥3)으로 positivity 실패가 *체계적*임을 보임.
  - `like≥3` = **교과서급 함정 figure**: naive +25.7pp가 IPTW 후에도 +25.0pp → 순진하면 "강건한 인과"로 오독.
    실제론 propAUC=1.00·극단가중으로 **positivity가 깨져 보정이 무의미**한 것. ("보정 후에도 안 변하면 오히려 의심.")
  - ⚠️ **[F1·F2·F3 정직성]**: 4레버 지도·like 함정 수치는 현재 **W=7 탐색값(일회성, 미커밋)**. 헤드라인으로 쓰려면 **M2에서
    재현 파이프라인을 커밋해야 함** — `impact.py`는 `aha_cart` 단일 레버에 하드코딩(`src/impact.py:29-58`)이라
    "머신 수정 0"이 아니다. 실제 M2 = ①레버 3종 인코딩(`personperiod.py`) ②`impact.py` 레버명 파라미터화
    ③config W=7 잠금(M0/D-11) ④레버별 positivity 재계산 ⑤`identifiability_map.py`→`docs/causal_report.md` ⑥Makefile 타깃+smoke.
    그 전까지 4레버는 "탐색적 결과"로 라벨, **헤드라인 단독 근거 = cart 단독(`make` 재현)**.
- **[F4] 결론(건설적 프레임)**: "레버가 없다"(막다른 길)가 아니라 → **"관측 로그로는 리텐션 레버를 *식별할 수 없다* —
  그래서 다음 단계는 추측이 아니라 *실험*이고, 그 설계가 4단계다."** (음성이 아니라 *실험으로 가는 진단*.)

### 4. A/B 설계 — "그럼 뭘 실험하나" (결론 = 설계)
- 3단계가 *왜* A/B를 강제하는지에서 직결. 가설(예: 첫 세션 폭-넓히기 넛지 → D7 재방문) · 랜덤화 단위(유저) ·
  주지표(D7 재방문) · **가드레일 + SRM(표본비 불일치) 체크 + MDE/표본수 표**(측정 base rate 기반).
- 산출 `docs/ab_test_design.md`. **인사이트**: 분석가의 일은 "정답 단정"이 아니라 *올바른 실험 설계*.

## 방법론 부록 (깊이·범위 증명 — 헤드라인 아님)
- **추천 폭(범위 증명, 실코드)**: 세션 prefix → 다음행동/카테고리 예측, POP→**마르코프**(c1 0.59→0.85 = "세션 직전 신호 > 인기";
  `version2/eval_baselines.py` 실측). GRU4Rec·SASRec·item-level은 *연구 노트*로만(C2C 73% 1회성 콜드스타트). **여기서 멈춤(YAGNI).**
- **OPE(차별화 한 문단, 코드 미구현·서술만)**: 합성에선 로깅정책 설계로 IPS/DR 무편향 *증명* 가능 / 실데이터는 노출
  propensity 부재로 식별 불가 — "노출 무작위성(A/B) 주어지면 다음 단계."

## 스코프·마일스톤 (1개월 — 정직판)
**MVS = 1·2·3단계 + 4단계 설계.** 부록은 깊이용. 새 산출물(funnel·cohort·causal·ab_test 리포트)은 현재 repo에 없음 → 신규 작성 비용 명시.
- **M0** 통합/게이트: `validate()` 확장 + **config W=7 잠금(D-11)** + v1 PLAN을 부록으로 흡수.
- **M1** 퍼널·코호트(번역, 데이터에 이미 있음) + *각 단계 양성 권고 1개*[F5].
- **M2 ★먼저 방어 가능하게★** 식별지도 재현 파이프라인 커밋(`identifiability_map.py`→`causal_report.md`) — 위 ①~⑥. *이게 해자를 주장→`make` 산출물로 바꾸는 한 수.*
- **M3** A/B 설계 문서. **M4** onepager(F6 훅)/통합 리포트.
- 잘라낸 것: 3막 OPE 코드 · GRU4Rec/SASRec · 대시보드 사양(전부 Tier 2/YAGNI).
