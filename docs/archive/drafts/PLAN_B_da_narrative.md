# 후보 B — DA 서사로 새로 쓰기
> **패션 C2C 그로스 분석 — 퍼널·리텐션·실험으로 "무엇을 키워야 하나"에 답하기 (DA 포트폴리오)**

## 정체성
"패션 중고 C2C 행동로그로 **퍼널 누수를 찾고, 리텐션 드라이버를 검증하고, 다음 실험을 설계하는** 프로덕트 데이터 분석가.
*겉보기 좋은 지표에 속지 않고* 진짜 북극성(재방문)을 움직이는 행동만 골라낸다."

→ **헤드라인 4개가 곧 작품**: 퍼널 → 코호트 리텐션 → 허영지표 적발 → A/B 설계. 모두 DA 채용 60초 스캔에 직격.
생존분석·g-computation·OPE는 **방법론 부록**(엄밀성 깊이 증명).

## 데이터 (단일 출처)
MerRec(Mercari C2C 행동로그) 5파티션·277만 이벤트·43k 유저·49만 세션·31일. 가입 필드 없음(→ t0=첫 관측).

## 분석 척추 (4단계 — DA-legible)

### 1. 퍼널 분석 — "어디서 새는가"
- 유저 도달률 view 98.7 → like 52.6 → cart 22.9 → offer 17.1 → **buy 4.4%**. 단계별 전환·절대 드롭오프.
- 세그먼트별(카테고리·세션길이·신규 vs 재방문) 퍼널 비교 → 최대 누수 지점 진단.
- 산출: `docs/funnel_report.md` + 퍼널 차트. **인사이트**: 전환 극희소(buy 4.4%) → 전환을 KPI로 쓰면 underpowered.

### 2. 코호트 리텐션 — "누가 돌아오는가"
- 첫 관측일 코호트 × 이후 재방문(D1·D3·D7…) 매트릭스 + 히트맵.
- 드라이버 진단: 재방문을 가르는 건 **인게이지먼트 폭**(세션수·활동일수·조회수)이지 단일 행동이 아님(SHAP).
- 산출: `docs/cohort_report.md` + 리텐션 히트맵. **인사이트**: 리텐션은 "한 번의 결정적 행동"이 아니라 폭에서 온다.

### 3. 허영지표 적발 — "보이는 레버가 진짜 레버인가" (Goodhart)
- 카트=리텐션 레버처럼 보임: 카트 유저 재방문 47.7% vs 32.7%(**+15pp 연관**).
- 인과 보정(g-formula/IPTW)에서 **~0으로 붕괴**(E-value 1.23). like≥3 등 4개 레버 모두 같은 함정.
- 산출: `docs/vanity_metric_finding.md`. **인사이트**: "큰 연관 ≠ 인과". 순진하게 카트 캠페인 돌리면 돈만 태움.

### 4. A/B 설계 — "그럼 뭘 실험해야 하나" (결론)
- 식별 불가가 *결론* → 진짜 답은 실험으로만. 가설(카트-넛지 → 재방문)·랜덤화 단위(유저)·주지표(D7 재방문)·
  가드레일·MDE/표본수(측정 base rate 기반).
- 산출: `docs/ab_test_design.md`. **인사이트**: 분석가의 일은 "정답 단정"이 아니라 "올바른 실험 설계".

## 방법론 부록 (깊이 — DS 머신을 "엄밀성 증명"으로)
3·4단계의 엄밀성을 받치는 기계: **이산시간 경쟁위험 생존**(검열·좌측절단 처리) · **g-computation/IPTW/MSM**(교란 보정,
E-value 봉투) · **OPE 합성 증명**(IPS/DR 무편향). "퍼널/코호트는 누구나 그리지만, *그 위에 인과·실험 설계까지* 얹는다."

## 정직성 규율 (승계)
관측→인과 단정 금지 · 가정 E-value 봉투 · 사전등록 vs 여정 분리 · config 단일출처 · 합성 백스톱 보존 · null도 1급 결과.

## 스코프
**MVS = 1·2·3단계(퍼널·코호트·허영지표) + 4단계 A/B 설계.** 부록(생존/OPE)은 깊이 증명용 Tier 2.
1개월: 1·2단계는 이미 데이터에 있음(번역) · 3단계는 v1 머신 재사용 · 4단계는 설계 문서.
