## 1. 분석 질문 (estimand 명시)

1. **(1막)** 세션 prefix로 **다음 행동타입 / 다음 product_id(=brand×c2 버킷) / c1**를 얼마나 맞히나?
   estimand = `P(e_{t+1} | e_{≤t})`. 평가 = 시간외 Recall@K·NDCG@K·MRR. *(item_id 레벨은 73% 1회성·중첩 18.9%로 사실상 불가 → 후보생성 한정.)*
2. **(2막)** 초기 윈도우에 **관측 행동 X(cart≥1회)** 를 한 코호트의 재방문(행동 리텐션 *프록시*)이, 안 한 코호트 대비
   *인과적으로* 높은가? estimand = `E[Y(do(X=1)) − Y(do(X=0))]` — **관측된 X의 조건부 do-효과**(추천 정책 효과 아님, §5 다리 주의).
   Y=embargo 이후 재방문(ΔCIF). 식별 = 순차교환가능성+positivity+consistency, E-value 봉투.
   🔬 **예비결과(2026-06-25): positivity near-1 꼬리로 식별 경계, 효과 DIVERGE → R2 폴백(연관+E-value) 정당화.**
3. **(3막)** 추천 정책 π의 가치 `V(π)=E_{a~π}[r]` 을 로그만으로 추정할 수 있나?
   **합성에선 *증명***(로깅정책 설계로 propensity 알려짐 → IPS/DR 무편향 수렴, 데모 확인). **실데이터에선 *식별 불가*** —
   유기적 로그라 노출 propensity 부재 → 3막 실데이터의 *결과는 점추정이 아니라 "OPE 적용 한계의 증명"*(§7).

## 2. 확정 설계 (락)

- **스파인 = 세션 기반 시퀀스 추천(1막).** prefix → next-action(인텐트, 5-클래스) + next-item(product_id=brand×c2 / c1 랭킹) 2-헤드.
  서술 동반자 = POP/마르코프/Session-KNN 베이스라인. 정체성·단독 방어선. *(item_id 추천은 콜드스타트 벽 → 연구 대상, §4.)*
- **계측층 = 인과 검증(2막).** v1 이산시간 경쟁위험 해저드 + g-formula/IPTW/MSM을 **관측 레버 X(cart)의 재방문
  인과효과 검증**으로 재배치. ⚠️ **predict≠cause 봉인**: 추천기는 X를 *예측·증폭*할 뿐 *유발*하지 않는다. 2막 estimand는
  *관측된* X의 조건부 do-효과이며 이를 추천정책 효과로 동일시하지 않는다. **1막↔2막 연결은 인과 사슬이 아니라
  "프록시 KPI가 북극성과 인과적으로 무관할 수 있다"는 경고 사례(Goodhart)** — 이 톤으로만 잇는다.
- **봉합층 = 오프폴리시 평가(3막), 이원화 — *적용 한계의 증명*.**
  - **합성(`_synth.py`)**: 로깅 정책 π₀를 *우리가 설계* → propensity 알려짐 → IPS/SNIPS/DR이 심어둔 V(π)를
    복원함을 **증명**(편향·분산 곡선, 데모로 무편향 수렴 확인). 3막의 *양성 핵심 결과*. **단 기존 GAMMA_TRUE 처치효과
    백스톱과는 별개 estimand → `_synth.py`에 정책/액션/보상 채널을 신규 추가**(2막용 백스톱은 보존).
  - **실데이터**: 유기적 로그라 노출 propensity가 **없음 → 고전 IPS 식별 *불가***. behavior-cloning π̂₀는 노출
    무작위성이 아니라 관측 예측확률이므로 IPS 무편향 조건(무교란 노출)을 충족 못 함(우회 아님, 순환논법). →
    **결과 = "식별 불가의 증명"**: BC-π̂₀+DR 민감도 스윕이 가정에 좌우됨을 *보이는 것*이 산출. 진짜 정책평가는 A/B 필요 → W2로 이관.
- **시간외 검증이 신뢰의 핵심.** 세션 *시작시각* 기준 시간분할. 랜덤 분할 금지(미래 누수).
- **카탈로그 차원 축소.** 1차 타깃 = `product_id`(brand×c2 18.3만 *버킷*), 보조 = `c1`(269). item_id 180만은 후보생성에만.
- **config 단일출처(v1 규약 승계).** 모든 가정·하이퍼파라미터는 `config/config.yaml`. 코드 매직넘버 금지(§6 W 봉합 참조).
- **정직성 규율 승계.** 관측→인과 단정 금지. 식별가정 명시 + E-value/민감도. **헤드라인 = 1막 추천(상대개선) +
  2막의 *정직한 음성*(허영지표 적발).** 양성 결과를 강요하지 않음 — null도 1급 결과.

### 폐기/강등
- v1 헤드라인("카트 효과 null")은 **2막 내부 발견으로 강등** — 작품 결론이 아니라 검증 절차의 산출물.

## 4. 방법론 — 1막 (MVS — 추천 *상대개선* + C2C 콜드스타트 *문제 연구*)

> 가능한가"라는 콜드스타트 문제 자체를 정량화**. 절대 Recall을 헤드라인으로 내세우지 않음. (M1 실측이 강도 확정.)

### 4.1 데이터 정형 & 누수 차단 평가 프로토콜 (← 리뷰 #2 수정)
- session → `[e_{t0}, …, e_{tn}]`. 피처 = 행동타입·product_id·c0/c1·brand·price-bucket·item_condition·dwell(Δstime). **past-only**.
- **분할(2단계, 누수 0):**
  1. **시간분할**: 세션을 *시작시각*으로 정렬 → train = day 1–24, valid = day 25–27, **test = day 28–31**.
     (train 시점 이후 정보는 모델·임베딩·인기통계 어디에도 유입 금지.)
  2. **세션 내 leave-last-out**: test 세션에서 **마지막 이벤트만 타깃**, 그 앞 prefix만 입력.
- 인기/co-visitation 통계는 **train 구간에서만** 집계(test 누수 방지).

### 4.2 타깃 2-헤드
- **(a) next-action type**: 다음 이벤트 ∈ {view, like, cart, offer, buy}. 인텐트·퍼널 신호. 클래스 불균형 → macro-F1 병기.
- **(b) next-item ranking**: 다음 product_id를 후보 풀에서 랭킹.

### 4.3 모델 사다리 (약→강, 각 단계가 다음의 베이스라인)
POP(인기) → **마르코프 1차**(v1 운영층 재사용) → **Session-KNN** → **GRU4Rec** → **SASRec/Transformer**.
콜드스타트: product_id 미관측 시 (c1·brand·price-bucket) **콘텐츠 임베딩**으로 폴백.

### 4.4 후보생성 → 랭킹 2단계 & 평가 편향 (← 리뷰 #5)
- 후보생성: co-visitation(같은 세션 공동출현) + 카테고리 인기 Top-N(예: N=500).
- **평가 분모 명시**: 180만 풀랭킹 불가 → **full-catalog over product_id/c1** 우선.
  sampled-negative 사용 시 **popularity-corrected**(Krichene–Rendle 2020 편향 보고) 병기.
- 지표: Recall@{10,20}·NDCG@{10,20}·MRR·HitRate@{10,20}. 서술 동반자 = 인기 대비 lift, c0별·세션길이별·콜드 vs 웜.
- **정직성**: 절대 Recall은 카탈로그 크기에 좌우 → **인기 베이스라인 대비 상대 개선**을 헤드라인으로.

### 4.5 🔬 베이스라인 실측 (log 2026-06-25, n=20k 페어, leave-last-out) — 헤드라인 근거
| 타깃(카탈로그) | POP R@20 | **Markov-1 R@20** | NDCG@20 | 판정 |
|---|---|---|---|---|
| item_id (1.5M, 199-neg) | 0.081 | **0.084** | 0.042 | 🔴 랜덤(0.10) 이하 → **item 추천 불가**(73% 1회성). 후보생성/연구문제로만 |
| product_id=brand×c2 (163K) | 0.103 | **0.470** | 0.325 | 🟡 warm 견인(cold=0.00) — *세션 brand 관성*이지 진짜 추천 아님. 단독 헤드라인 약함 |
| c1 (297) | 0.590 | **0.854** | 0.670 | 🟢 견고(작은 출력공간 + 세션 카테고리 일관성). 헤드라인 가능 |
| next-action(5클래스) | acc 0.80 / **macro-F1 0.15** | acc 0.81 / **macro-F1 0.30** | — | acc는 view 80%로 무의미 → **macro-F1만** |

- **살아있는 발견(1막 자산)**: POP→Markov 게인(product 0.10→0.47, c1 0.59→0.85) = **"세션 직전 신호 > 인기"**. 이게 1막의 진짜 결과.

## 5. 방법론 — 2막 (인과 검증, v1 자산 재배치)

- **처치 X 조작적 정의(← 리뷰 #3)**: 초기 윈도우 `[t0, t0+W)` 안에 **`add_to_cart` ≥ 1회**(주 레버) / 보조 `item_like` ≥ k회.
  *관측된* 행동이며 추천기 산출물이 아님. ⚠️ **다리 약화 명시**: 추천기는 X를 *예측·증폭*할 뿐 *유발*하지 않으므로, estimand는
  "추천 정책 효과"가 아니라 **관측 X의 조건부 do-효과**다. 1막↔2막은 인과 사슬이 아니라 *Goodhart 경고 사례*로만 연결.
- **머신 재사용**: 이산시간 경쟁위험 해저드(`survival.py`) + g-formula(주력)/IPTW·MSM(교차) + naive 대조(`impact.py`).
  🔬 코드 레벨 확인: X=cart는 `personperiod`에 `aha_cart`로 이미 인코딩 → **수정 없이 재사용**, config W만 오버라이드.
- **결과 Y**: embargo 이후 재방문(행동 리텐션 *프록시*). competing risk = 이탈. 좌측절단=지연진입.
- **🔬 예비결과(log 2026-06-25, W=7 실측)**: naive Δ=+0.206 → IPTW Δ=+0.013(clip0.99) → g-comp Δ=**−0.035**.
  positivity **e_max=1.0**(고활동 유저서 cart 거의 결정적, near-1 꼬리), SMD 0.37–0.78(전부 불균형, 활동량이 cart의 교란),
  진단 **DIVERGE**(abs-diff 4.7pp>3pp). E-value: naive RR1.62→2.63, g-comp RR0.90→**1.45**(약한 미관측 교란이면 설명 소멸).
  → **결론: 카트는 인과 레버가 아니라 활동량 마커.** 2막은 *허영지표 적발*을 정직한 결과로 보고(양성 강요 금지).
- **프록시-북극성 간극 정량화**: 1막 프록시(다음행동 CTR)와 북극성 프록시(재방문)의 **상관 vs 인과 차이** →
  "프록시는 오르는데 북극성은 안 움직이는 행동" 식별(Goodhart). v1의 "카트=마커" 결론이 여기 산다.
- **봉투·승계**: 식별가정 명시 + **E-value** + **절대차 가드**(`decisions.md` A-2 승계)로 null-스케일 착시 구분.
  v2 2막은 **`decisions.md` A-1(전환→리텐션 승격)·A-2(절대차 가드)·D-5(조정집합서 n_cart 제외) 전제 승계**.

## 7. 방법론 — 3막 (오프폴리시 평가 *및 그 식별 한계의 증명* ← 리뷰 #1 핵심)

> 3막의 산출은 *양성 V(π)*가 아니라 **"OPE를 어디까지 적용할 수 있는가의 경계"**다. 합성=무편향 증명, 실데이터=식별 불가 증명.

### 7.1 합성(`_synth.py`) — *증명* (🔬 데모로 무편향 수렴 확인됨)
- **신규 구조 필요**: 3막 합성은 2막과 *다른 estimand*(정책가치 V(π))다. `_synth.py`에 (a) 설계된 로깅정책 π₀(a|x),
  (b) 액션의존 보상 r(x,a), (c) 참 V(π)를 **신규 추가**. ⚠️ 기존 `GAMMA_TRUE` 처치효과 백스톱은 *2막용*이므로 보존(재사용 아님).
- IPS·SNIPS·Direct Method·**Doubly-Robust**가 심어둔 V(π)를 복원하는 **편향·분산·MSE 곡선** 제시.
  🔬 데모: π₀ 알려질 때 편향 n=500 ±0.05 → n=128k ±0.003 수렴 — §7.1 실현 가능.
- 동일 합성에서 g-formula도 ΔCIF 복원 → **2막·3막 추정기 정확성의 공동 보증서**.

### 7.2 실데이터 — *식별 불가의 증명* (점추정 아님)
- MerRec은 유기적 로그 → **노출 propensity 부재 → 고전 IPS/SNIPS 식별 *불가***(반사실 노출 채널이 없음).
- behavior-cloning π̂₀는 노출 무작위성이 아니라 *관측 예측확률*이라 IPS 무편향 조건(무교란 노출)을 충족 못 함 → **우회가 아니라 순환논법**.
- **결과 = 식별 불가의 *증명***: BC-π̂₀+DR을 *돌려서* 민감도 스윕이 가정에 좌우/비강건함을 **보이는 것**이 1급 산출(음성의 가치).
  ⚠️ "강건구간"으로 보고하지 않음 — 구간이 좁아도 그건 *분산*이 작은 것이지 *편향(식별 결함)*이 작다는 보증이 아니다.
- 진짜 정책가치 추정엔 노출 무작위성(A/B)이 필요 → **W2(A/B 설계 사양)로 이관**. (R4를 사전등록 예상결과 D-12로 잠금, §13.)
