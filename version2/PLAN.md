# v2.1 — 세션 추천 × 인과 검증 × 오프폴리시 평가 (3막 구조)

> **정체성**: "추천을 만들되 그것을 **헤드라인 지표(CTR·Recall)** 가 아니라 **북극성 지표(리텐션)** 로 검증하는 그로스/추천 분석가 —
> 시퀀스 추천을 만들고(1막), 추천이 *키우려는 행동*이 진짜로 리텐션을 올리는지 인과로 따지고(2막),
> 추천 정책을 온라인 A/B 없이 평가하는 오프폴리시 추정기를 합성 ground-truth로 검증한다(3막)."
>
> v1(인과 리텐션 단독)의 약점 — *데이터가 인과를 못 받쳐 헤드라인이 null 음성 결과에 묶임* — 을, 데이터의
> **native 강점(세션 시퀀스·180만 카탈로그)** 위에 인과 머신을 **검증 도구로 재배치**해 해소한다.
> v1 자산(`survival.py`·`impact.py`·`_synth.py`·`config.yaml`)을 2·3막으로 승계한다.
>
> **v2→v2.1 변경(이 리뷰의 결과)**: ①3막 OPE의 propensity 부재(식별 결함)를 "합성=증명/실데이터=추정+민감도"로
> 이원화 ②1막 평가 누수 프로토콜 정밀화 ③2막 estimand 조작적 정의 ④Phase-0 GO 임계 정량화 ⑤카탈로그 평가

## 1. 분석 질문 (estimand 명시)

1. **(1막)** 세션 행동 시퀀스 prefix로 **다음 행동타입 / 다음 product_id**를 얼마나 맞히나?
   estimand = `P(e_{t+1} | e_{≤t})`. 평가 = 시간외 Recall@K·NDCG@K·MRR.
2. **(2막)** 초기 윈도우에 **행동 X(예: cart≥1회)** 를 한 코호트의 재방문 리텐션이, 안 한 코호트 대비
   *인과적으로* 높은가? estimand = `E[Y(do(X=1)) − Y(do(X=0))]`, Y=embargo 이후 재방문(ΔCIF). 식별 = 순차교환가능성+positivity+consistency, E-value 봉투.
3. **(3막)** 추천 정책 π를 깔았을 때 보상(다음행동/리텐션)의 기대값 `V(π)=E_{a~π}[r]` 을 **로그만으로** 추정할 수 있나?
   합성에선 *증명*(로깅정책 알려짐), 실데이터에선 *추정 propensity + 민감도*(식별 한계 명시).

## 2. 확정 설계 (락)

- **스파인 = 세션 기반 시퀀스 추천(1막).** prefix → next-action(인텐트, 5-클래스) + next-item(product_id 랭킹) 2-헤드.
  서술 동반자 = POP/마르코프/Session-KNN 베이스라인. 정체성·단독 방어선.
- **계측층 = 인과 검증(2막).** v1 이산시간 경쟁위험 해저드 + g-formula/IPTW/MSM을 **추천이 키우려는 레버 X의
  리텐션 인과효과 검증**으로 재배치. 처치 X는 *관측된 초기행동*(아래 §4.2 조작적 정의), 추천기와의 연결은
  "추천이 최적화하는 프록시 = X의 발생확률"이라는 **논리적 다리**(추천은 X를 예측·증폭하려 하지, 직접 유발하지 않음).
- **봉합층 = 오프폴리시 평가(3막), 이원화.**
  - **합성(`_synth.py`)**: 로깅 정책 π₀를 *우리가 설계* → propensity 알려짐 → IPS/SNIPS/DR이 심어둔 V(π)를
    복원함을 **증명**(편향·분산 곡선). 이것이 3막의 *양성 핵심 결과*.
  - **실데이터**: 추천 로그·노출 propensity가 **없음**(유기적 행동 로그) → 고전 IPS **식별 불가**.
    → 행동정책 π₀를 **추정**(behavior cloning)하고 **DR**(모델편향 완화) + clipping + **민감도(propensity 오설정 스윕)**.
    한계를 1급 결과로 명시. "온라인 A/B 없이 정책 비교"는 *합성에서 검증된 절차를 실데이터에 조심스럽게 적용*.
- **시간외 검증이 신뢰의 핵심.** 세션 *시작시각* 기준 시간분할. 랜덤 분할 금지(미래 누수).
- **카탈로그 차원 축소.** 1차 타깃 = `product_id`(18.3만), 보조 = `c1`(269). item_id 180만은 후보생성에만.
- **config 단일출처(v1 규약 승계).** 모든 가정·하이퍼파라미터는 `config/config.yaml`. 코드 매직넘버 금지.
- **정직성 규율 승계.** 관측→인과 단정 금지. 식별가정 명시 + E-value/민감도. 단 *헤드라인은 1막의 양성 결과*.

### 폐기/강등
- v1 헤드라인("카트 효과 null")은 **2막 내부 발견으로 강등** — 작품 결론이 아니라 검증 절차의 산출물.
- 좌측절단/가입부재 이슈는 2막에서만 등장(1막 추천은 가입 불요) → v1의 가장 아픈 약점이 1막에선 무력화.

## 4. 방법론 — 1막 (MVS, 양성 헤드라인)

### 4.1 데이터 정형 & 누수 차단 평가 프로토콜 (← 리뷰 #2 수정)
- session → `[e_{t0}, …, e_{tn}]`. 피처 = 행동타입·product_id·c0/c1·brand·price-bucket·item_condition·dwell(Δstime). **past-only**.
- **분할(2단계, 누수 0):**
  1. **시간분할**: 세션을 *시작시각*으로 정렬 → train = day 1–24, valid = day 25–27, **test = day 28–31**.
     (train 시점 이후 정보는 모델·임베딩·인기통계 어디에도 유입 금지.)
  2. **세션 내 leave-last-out**: test 세션에서 **마지막 이벤트만 타깃**, 그 앞 prefix만 입력.
- 인기/co-visitation 통계는 **train 구간에서만** 집계(test 누수 방지).

### 4.2 타깃 2-헤드
- **(a) next-action type**: 다음 이벤트 ∈ {view, like, cart, offer, buy}. 인텐트·퍼널 신호. 클래스 불균형 → macro-F1 병기.
- **(b) next-item ranking**: 다음 product_id를 후보 풀에서 랭킹.

### 4.3 모델 사다리 (약→강, 각 단계가 다음의 베이스라인)
POP(인기) → **마르코프 1차**(v1 운영층 재사용) → **Session-KNN** → **GRU4Rec** → **SASRec/Transformer**.
콜드스타트: product_id 미관측 시 (c1·brand·price-bucket) **콘텐츠 임베딩**으로 폴백.

### 4.4 후보생성 → 랭킹 2단계 & 평가 편향 (← 리뷰 #5)
- 후보생성: co-visitation(같은 세션 공동출현) + 카테고리 인기 Top-N(예: N=500).
- **평가 분모 명시**: 180만 풀랭킹 불가 → **full-catalog over product_id(18.3만)** 우선.
  sampled-negative 사용 시 **popularity-corrected**(Krichene–Rendle 2020 편향 보고) 병기.
- 지표: Recall@{10,20}·NDCG@{10,20}·MRR·HitRate@{10,20}. 서술 동반자 = 인기 대비 lift, c0별·세션길이별·콜드 vs 웜.
- **정직성**: 절대 Recall은 카탈로그 크기에 좌우 → **인기 베이스라인 대비 상대 개선**을 헤드라인으로.

## 5. 방법론 — 2막 (인과 검증, v1 자산 재배치)

- **처치 X 조작적 정의(← 리뷰 #3)**: 초기 윈도우 `[t0, t0+W)` 안에 **`add_to_cart` ≥ 1회**(주 레버) /
  보조 `item_like` ≥ k회. *관측된* 행동이며 추천기 산출물이 아님. 추천기와의 다리 = "추천이 최적화하는 프록시 KPI =
  X의 발생확률" → **추천이 X를 키웠을 때 북극성이 따라 오르는지**를 묻는 것.
- **머신 재사용**: 이산시간 경쟁위험 해저드(`survival.py`) + g-formula(주력)/IPTW·MSM(교차) + naive 대조(`impact.py`).
- **결과 Y**: embargo 이후 재방문(리텐션). competing risk = 이탈. 좌측절단=지연진입.
- **프록시-북극성 간극 정량화**: 1막 프록시(다음행동 CTR)와 북극성(재방문)의 **상관 vs 인과 차이** →
  "프록시는 오르는데 북극성은 안 움직이는 행동" 식별(Goodhart 주석). v1의 "카트=마커" 결론이 여기 산다.
- **봉투**: 식별가정(순차교환가능성·positivity·consistency) 명시 + **E-value**. **절대차 가드**(v1 진단 프로토콜 승계)로
  null-스케일 착시(분모≈0) 구분.

## 7. 방법론 — 3막 (오프폴리시 평가, 이원화 ← 리뷰 #1 핵심)

### 7.1 합성(`_synth.py`) — *증명*
- 로깅 정책 π₀와 보상모델을 *우리가 설계* → 참 V(π)와 propensity 알려짐.
- IPS·SNIPS·Direct Method·**Doubly-Robust**가 심어둔 V(π)를 복원하는 **편향·분산·MSE 곡선** 제시.
- 동일 합성에서 g-formula도 심어둔 ΔCIF 복원 → **2막·3막 추정기 정확성의 공동 보증서**.

### 7.2 실데이터 — *추정 + 민감도*(식별 한계 1급 보고)
- MerRec엔 추천 노출 propensity가 **없다** → 고전 IPS 식별 불가(이 사실을 결론에 명시).
- 행동정책 π₀ **추정**(behavior cloning: 1막 모델을 로깅정책 근사) → **DR**(모델편향 완화) + propensity **clipping**.
- **민감도**: propensity 오설정 스윕·clipping 임계 스윕으로 V(π) 추정의 강건구간 보고.
- 결론: "합성에서 검증된 OPE 절차를 실데이터에 *조심스럽게* 적용 — 점추정이 아니라 강건구간으로."

---

## 8. 스코프 규율 (멈추는 선 — 락)
- **MVS(단독 완결작) = Phase0 게이트 → 1막 + 2막.** ← 멈추는 선.
  - **1막 [Tier 1]**: POP→마르코프→GRU4Rec, 시간외 Recall@K/NDCG (SASRec은 Tier 2).
  - **2막 [Tier 1]**: 레버 X의 리텐션 인과효과, g-formula + E-value.
- **3막 OPE [Tier 2 확장, MVS 비포함]**: 합성 OPE 증명(§7.1)을 우선 차별화로, 실데이터 DR/민감도(§7.2)는 그 다음.
  *합성 증명은 2막 추정기 보증서로도 쓰여 가치 높으나, 작품의 단독 완결은 1·2막으로 성립.*
- **그 외 Tier 2**: SASRec 콘텐츠 하이브리드 · IPTW/MSM 교차 · next-action↔next-item 멀티태스크.
- **Tier 3**: 오퍼/협상(가격) 모델 곁가지 · 유저-FE · 시간외 백테스트.

## 10. 리스크 레지스터 & kill 기준 (← 리뷰 #7)
- **R1 콜드스타트 과다**(중첩 <50%): → c1-레벨 타깃 + 콘텐츠 폴백. kill 아님.
- **R2 2막 positivity 실패**(처치/대조 <500): → **인과 주장 철회, 연관+E-value만**(v1 규약). 헤드라인은 1막이라 작품은 생존.
- **R3 SASRec 컴퓨트 초과**: → GRU4Rec까지가 MVS, SASRec은 Tier2. (CPU/단일GPU 가정, 학습시간 config로 제한)
- **R4 OPE 실데이터 발산**: → 합성 증명을 헤드라인, 실데이터는 "식별 한계 사례"로 보고(음성도 정직한 결과).
- **재현성**: 시드 고정·환경 핀(`requirements.txt`)·`make all` 1커맨드 재생성. 네트워크 없이 `make test`(합성).

## 12. 평가 KPI (포트폴리오 셀링포인트)
- **1막**: 인기 대비 Recall@20·NDCG@20 상대 개선(시간외), 콜드 vs 웜·세션길이 robustness.
- **2막**: 프록시 vs 북극성 간극을 인과로 보인 사례(E-value 봉투) — "추천 엔지니어의 인과 리터러시".
- **3막**: OPE 추정기가 합성 ground-truth 복원(편향·분산 곡선) → 정책 비교를 *온라인 A/B 없이* + 실데이터 식별 한계의 정직한 보고.
